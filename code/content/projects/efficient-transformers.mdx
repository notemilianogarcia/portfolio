# RAG Benchmark Service

## What I Built

A production-grade **Retrieval-Augmented Generation system** for scientific question answering. The project prioritizes reproducibility, rigorous evaluation, and ML engineering best practicesâ€”starting with BM25 baselines before scaling to dense retrieval.

## Impact & Metrics

- **64 unit tests** with **93% code coverage**
- **BM25 baseline** evaluation across Recall@5/10/50, MRR, nDCG, MAP
- **Artifact versioning** for reproducible experiments
- **SciFact dataset** integration (5,183 documents, 300 test queries)
- **Production-ready** architecture with clear MLops path

## Technical Architecture

**Data Pipeline:** Schema validation, referential integrity checks, statistics reporting  
**Indexing:** BM25 with timestamp-based versioning and pointer-based discovery  
**Retrieval:** Modular top-k search with fallback mechanisms  
**Evaluation:** Offline metrics (Recall, MRR, nDCG, MAP) with JSON/Markdown reports  

## Stack

| Component | Technology |
| --- | --- |
| **Language** | Python 3.11+ |
| **Retrieval** | rank-bm25 (BM25) |
| **Testing** | pytest (93% coverage) |
| **Quality** | Ruff, mypy, Black |
| **Tracking** | MLflow |
| **Datasets** | BEIR (SciFact) |
| **Planned** | sentence-transformers, FAISS, FastAPI |

## Key Decisions

1. **Baseline-first:** Start with interpretable BM25 before neural methods
2. **Offline-first:** Rigorous evaluation gates before production
3. **Modular design:** Clean separation across data, indexing, retrieval, evaluation layers
4. **Reproducibility:** All artifacts and reports are versioned
5. **Developer experience:** Makefile workflows, comprehensive testing, clear documentation

## Roadmap

- **Milestone 1:** Data pipeline + BM25 baseline (COMPLETED)
- **Milestone 2:** Dense retrieval (sentence-transformers + FAISS)
- **Milestone 3:** Advanced evaluation (hyperparameter sweeps, dashboards)
- **Milestone 4:** FastAPI service layer
- **Milestone 5:** Production ops (monitoring, drift detection, CI/CD)

## Why This Matters

This project demonstrates **ML engineering discipline**: reproducibility through versioning, rigor through offline metrics, and sustainable architecture through modularity. The foundation is built to scale from baseline to production RAG systems.
